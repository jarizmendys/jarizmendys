# 🐳 Docker Desktop + WSL

## 1. 🏗️ Infraestructura y Virtualización (Docker)

 Docker proporciona aislamiento de procesos mediante contenedores, permitiendo la paridad de entornos entre desarrollo y producción.

### 🔌 Integración WSL2 desde Docker Desktop

Para asegurar la comunicación entre Docker Desktop y el kernel de Linux, siga la ruta de configuración:

1. Acceda a **Settings** > **Resources** > **WSL Integration**.
2. Habilite el toggle **Enable integration with my default WSL distro**.
3. Active manualmente la integración para las distribuciones instaladas (**Ubuntu** y/o **Debian**).

### 🖼️ Gestión de Imágenes

Administración del ciclo de vida de artefactos de software:

```bash
# 🚀 Despliegue y Construcción
docker build .                              # Construir una imagen 
docker build -t [imagen] .                  # Construir una imagen, nombre imagen en directorio actual.
docker build -t [image]:[latest] .          # Construir una imagen, nombre imagen y Version.
docker image tag [viejo:tag] [nuevo:tag]    # Renombrar o versionar una imagen.
docker load --input [proyecto.tar]          # Importar una imagen desde un archivo .tar

# 📦 Consulta y Portabilidad
docker images                               # Listar imágenes locales
docker images -a                            # Listar todas las imágenes (incluyendo capas intermedias)
docker save jhos/proyecto > proyecto.tar    # Exportar una imagen a un archivo comprimido

# 🧹 Finalización y Limpieza
docker rmi [nombre_imagen]                  # Eliminar una imagen por ID o nombre
docker rmi -f [nombre_imagen]               # Forzar eliminación de imagen vinculada a contenedores
docker image prune                          # Eliminar imágenes (huérfanas)
docker image prune -a                       # Eliminar todas las imágenes sin contenedores asociados
```

### 📦 Gestión de Contenedores

Control de instancias en tiempo de ejecución:

```bash
# 🚀 Despliegue y Ejecución
docker run --name [nombre] [imagen]         # Ejecuta y nombrar un contenedor
docker run -d -p [host]:[cont] [imagen]     # Ejecutar en segundo plano con mapeo de puertos
docker run -it [imagen] [comando]           # Ejecutar en modo interactivo (ej. bash)
docker run --rm [imagen]                    # Ejecuta un contenedor y eliminarlo automáticamente al detenerse
docker run -v [ruta_host]:[ruta_cont] [img] # Ejecuta un contenedor vinculando un volumen para persistencia de datos
docker start [id/nombre]                    # Iniciar un contenedor detenido
docker rename [viejo] [nuevo]               # Modificar el alias de un contenedor

# 🩺 Consulta y Control
docker ps                                   # Listar contenedores activos
docker ps -a                                # Listar Contenedores (activos/detenidos)
docker ps -a --filter "ancestor=[imagen]"   # Filtrar por imagen de origen
docker pause [contenedor]                   # Suspender los procesos internos
docker stop [id/nombre]                     # Detener la ejecución de forma segura

# 🧹 Finalización y Limpieza
docker rm id [id/nombre]                    # Eliminar un contenedor específico
docker rm -f [id/nombre]                    # Forzar eliminación de contenedor activo 
docker rm -f $(docker ps -aq)               # Borrar todo y forzado de todas las instancias
docker container prune                      # Borrar todos los Contenedores los Detenidos
```

### 🌐 Inspección, Redes y Volumes

Para depurar contenedores y gestionar la comunicación entre ellos, utilizamos:

```bash
docker inspect web                                 # Ver toda la configuración del contenedor "web"
docker network ls                                  # Listar redes (bridge, host, none)
docker network create red-local                    # Crear una red personalizada
docker run -d --name app --net red-local [imagen]  # Ejecutar un contenedor en una red específica
docker volume ls                                   # Lista todos los volúmenes de datos creados en el sistema.
docker volume create [nombre]                      # Crea un volumen independiente para persistencia de archivos.
docker volume rm [nombre_volumen]                  # Eliminar un volumen específico
docker volume prune                                # Elimina todos volúmenes que no están siendo usados por contenedor.
```

💡 **Ejecución de Referencia:**

- `docker run -it --rm -d -p 8080:80 --name web sitioweb`
- `docker run -it --rm -d -p 8085:80 --name [name][name_Imagen:tag]`
- `docker run -d -p 8086:8086 --name mi-nodo-app hello-node-app:latest`

🔍 **Análisis de Flags de Ejecución:**

- `-it`: Modo **Interactivo + TTY**. Permite la entrada directa por terminal al contenedor en vivo.
- `--rm`: **Auto-eliminar**. Destruye el contenedor al detenerlo, evitando residuos en el sistema.
- `-d`: **Detached**. Ejecuta el proceso en segundo plano, liberando la consola del host.
- `-p 8080:80`: **Mapeo de Puertos**. Vincula el puerto `8080` de Windows al puerto `80` interno del contenedor (ej. Nginx).
- `--name`: Asignación de un alias identificador para evitar el uso de hashes hexadecimales.

### 🛠️ Dockerfile vs Docker Compose (Conceptos y Ejemplos)

Aunque ambos trabajan en equipo, cumplen funciones totalmente diferentes en el ciclo de vida de tus aplicaciones:

- **Dockerfile (El plano de construcción - Individual):** Es un archivo de texto con instrucciones secuenciales para **construir una sola imagen** personalizada (ej. copiar código, instalar librerías). Comando clave: `docker build`.

- **Docker Compose (El director de orquesta - Multicontenedor):** Es un archivo en formato YAML (`docker-compose.yml`) que sirve para **definir y correr aplicaciones que necesitan múltiples contenedores** interconectados (ej. tu app + base de datos). Comando clave: `docker compose up`.

#### 📝 Ejemplos de un `Dockerfile` (Para una app de Node.js) y (Python)

# 5. Copiar el resto del código de la aplicación

# 6. Exponer el puerto en el que corre la app

# 7. Comando para iniciar la aplicación

#### 📝 Ejemplo de un `docker-compose.yml` (App + Base de Datos)

<details> 
<summary>YAML - 📂 Haz clic aquí para ver el archivo docker-compose.yml</summary>

```yaml
version: '3.8'

services:
  # Servicio 1: Nuestra aplicación web
  web:
    build: . # <--- Indica que use el Dockerfile del directorio actual
    ports:
      - "8080:3000" # <--- Mapea el puerto 8080 del host al 3000 del contenedor
    depends_on:
      - db # <--- Le dice a Docker que encienda primero la base de datos
    environment:
      - DB_HOST=db
      - DB_PORT=5432

  # Servicio 2: Base de datos PostgreSQL
  db:
    image: postgres:15 # <--- Descarga la imagen oficial directamente
    environment:
      POSTGRES_USER: usuario_admin
      POSTGRES_PASSWORD: mi_clave_secreta
      POSTGRES_DB: mi_base_de_datos
    volumes:
      - postgres_data:/var/lib/postgresql/data # <--- Para no perder los datos al apagarlo

volumes:
  postgres_data: # <--- Declaración del volumen persistente
```

</details>

### 🌐 Orquestación con Docker Compose

Para gestionar aplicaciones multicontenedor definidas en archivos `.yml`, utilizamos:

| **Comando**            | **Propósito Técnico**                                                  |
| ---------------------- | ---------------------------------------------------------------------- |
| `docker compose build` | Construye o reconstruye los servicios definidos en el archivo compose. |
| `docker compose up`    | Crea y levanta todos los contenedores, redes y volúmenes definidos.    |
| `docker compose down`  | Detiene y elimina los contenedores, redes e imágenes creadas por `up`. |
| `docker compose ps`    | Lista el estado de los servicios gestionados por el archivo actual.    |

```bash
# 🚀 Despliegue de servicios
docker compose build                         # Preparar imágenes de los servicios
docker compose up                            # Levantar la infraestructura completa
docker compose up -d                         # Levantar en segundo plano (Detached)
docker compose up --build                    # Forzar reconstrucción y levantar

# 🛑 Finalización
docker compose stop                          # Detener los servicios (sin borrarlos)
docker compose down                          # Limpiar todo lo creado por el archivo
docker compose logs -f                       # Ver logs de todos los servicios en tiempo real
```
