# 🌿 Control de Versiones (GIT)

## 1. 🛠️ Desarrollo de Software y Control de Versiones (Git)

### 🔄 Flujo de Trabajo Profesional

Siga esta secuencia para garantizar la integridad del control de versiones:

1. `git init`: Inicialización del seguimiento en el directorio local.
2. `git config --global user.name "Nombre"` y `git config --global user.email "correo@ejemplo.com"`: Configuración de la firma de autoría.
3. `git remote add origin [URL]`: Vinculación del repositorio local con el servidor remoto (GitHub).
4. `git remote -v`: **Verificación crítica** de la conectividad y el bridge construido.
5. `git add .`: Preparación del *staging area* (seguimiento de todos los cambios).
6. `git commit -m "Mensaje descriptivo"`: Creación del snapshot local.
7. `git push origin master`: Despliegue de los cambios a la nube.

*💡 Nota: La autenticación requiere un* ***Personal Access Token (PAT)*** *en lugar de la contraseña estándar.*
